1.1 Purpose

Develop a web application that allows authenticated users to manage the resources regarding the care of an animal or group of pets within a possible pet owner family.

1.2 Product scope

Manage the feeding schedule, restrictions, various multimedia resources (photos, movies, audio files) regarding the evolution of the animal, the medical history (including first aid methods), the relationship with other animals will be targeted.

2.1 Product functions

The user will be able to:

  • Upload images, videos about his/her personal pet/s and its evolution in time
  • Monitorise its eating habits, medical history
  • Discover other people which have the same interests or the same kind of pets
  • Post relevant moments on social media
  • Make the memories of the pets public, available only to the family or private depending on the preferences
  • Overall create an unique and personalized experience for your pets
  • Create families and add multiple pets to it
  • Visualize other pet's profiles and evolution

2.2 User classes and characteristics

  • Families
  • Pet/s owners

2.3 Operating environment

  • All the files and contents will be uploaded to a database which stores everything
  • The pages are designed to be used on various devices(phones, laptops, tablets) and operating systems

2.4 Design and Implementation Constraints

  • In order to give a full experience to the end user, the user interface will consist in HTML5, CSS3, JavaScript and the back-end of the application will consist in PHP scripts, SQL database
  • Only the interface scripts will be public, the back-end needs to be private so no user can alter the integrity of the website

2.5 User documentation

  • There is a dedicated section named first aid which includes a guide with precious information in case of a pet emergency.

2.6 Assumptions and Dependencies

  • Everything is server-based so it shouldn't be down for a long time when it crashes or has updates ongoing
  • The database should not have any issue with storing the data (storage capacity, file format, wrong places for files)

3.1 User interfaces

  • All types of accounts will have access to all of the pages that they are supposed to see (regarding private or public content)
  • The user will always be able to Log Out and the guest will be able to Register/Sign In from the pages
  • Each user will be able to have more pets and the pets will each have a profile of their own, while being able to make friends in the community
  • Users will be notified when any error will appear
  • Users will be able to report any problem to the admin account and the solution will come in a short time no matter the request
  • All registered users can add or remove significant memories of their pets, track their evolution and schedule, make posts private or public, post on social media and be free to have a profile configured to their own expectations

3.2 Hardware interfaces

  • All device types will be supported
  • The data will be stored on a server-based platform with enough storage to handle all media files
  • All files will be accessed by using scripts that are based on SQL commands

3.3 Software interfaces

  • The data is stored in a server-based database and the website will be able to access all needed documents via commands done by a script
  • The website will be accessible from all types of the devices as they will all be made responsive
  • Each and every type of file (video, audio, text) will be added to the needed table of the database via PHP scripts
  • All software components will have access to the required scripts or html/css templates as the pages need to be easy to use and effective in doing their tasks

3.4 Communications interfaces

  • All websites are on web browsers
  • All users are registered via e-mail addresses and will get any updates that are made from or to their account (report results, account deletion, new media added by friends, health care and appointments reminders etc)
  • HTTPS will be used, taking into considerations HTTP protocols regarding safety and general use

4.1 Generate pet profiles

  • Each family has the option to add new pets. On the family section, there is a button add pet which redirects the user to a panel which must be completed according to all the information (name, profile picture, restrictions, food planning - each day and birthday) needed for the pet. Each pet new created will be stored in the pet database. 

4.2 Find friends

  • The find friends option is available for the pets. Each pet has on its own profile the add friend button which will redirect the pet to all the other pets in order to begin a friendship. If the friendship is created, the friendship database will store each friend's id. Also, there is a button on each pet's profile to view the pet's friends. 

4.3 Post on social media

  • The user can share a post to facebook, linkedin and twitter. The posts he can share are the one of his own and its family. The sharing options opens a windows pop-up which redirects the post to the desired social media. An account is mandatory if someone wants to access the post shared. If the user wants to access the post shared, he will be redirected, if he has an account, to a page only with the post shared. 

4.4 Make memories public/only family/private

  • The memories have three levels of privacy: public, only family and private. The level of privacy is unique on every post and influences the way it appears for other users. The public privacy is the least restrictive one, a public post is available in the homepage and anyone can see it. The family only type of privacy restricts the post from being available on the homepage but still makes it public for the family members and the pet's friends. The private level of privacy only allows the post to be seen by the user who posted it. The security level is stored in the database as a section in the pet database. 

4.5 Manage health care and feeding schedule

  • The pet profile will include the possibility of setting an everyday of the week food plan for the pet. The plan can always be edited and will be stored in the pet database where there is a column for each day of the week. The medical history of the pet can be accessed through the medical history button which will redirect the user to a dedicated page for the medical history. There will be listed all the pet's medical issues and occurrences and more can be added through the add new entry button. Each medical fact must have a description and a date (on which it occurred ) and  are stored in a special medical history database. 

4.6 Login

  • All the users will have to introduce their credentials according to the username and password introduced in the register panel. Those are verified with the information in the database and if they're correctly introduced the user will be logged in, redirected to the homepage and a session for him will be created. 

5.1 Security Requirements

  • All data will be encrypted so the ones that want to make illegal changes will not be able to
  • Regarding the attacks, we have developed manners to handle successfully 2 types of attacks: 
  • 1. NO-SQL INJECTION: The definition of the concept is that all queries must be handled in scripts outside the html website. The scripts need all to be parameterised so that the users cannot create statements of their own and also need to be on scripts that are introduced in the html website with the "required" mark so that it needs to be executed that time and to be leaving in the view page only the output of the script. We have configured the scripts by the PDO variable, which reconnects to the database every time it needs to handle the database content.
  •  2. CROSS-SITE SCRIPTING: The definition of the concept is that hackers can insert into input fields from html websites malicious input that redirects to either websites of their own from which they do malicious things. We are protecting our website from this behavior by using htmlspecialchars(), a function of PHP that filters the input so that no signs like "<, >... " return the exact same output in the database. By doing this, the input is safe no matter the input so we can have the website secured as successfully as possible 

5.2 Software Quality Attributes

  • The profiles and pages will be adapted to the end user
  • The content will be available to the end users as the privacy requirements are satisfied
  • All data will be displayed accordingly
  • All content can be modified, deleted or new content can be added at any time
  • The website can be used on all types of devices, being reliable to all users
  • All content will be organised with no imperfections
  • The website will be able to be used by all users, taking into consideration that the interface is easy to understand and use
